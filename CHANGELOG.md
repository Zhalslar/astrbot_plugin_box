# 更新日志

## v1.4.0

新功能：

- 引入 `BoxService` 服务层，用于处理 QQ 资料查询、数据转换以及卡片图片渲染。
- 新增 `BoxUserProfile` 模型，用于规范化和格式化 QQ 用户数据，包括可选的外部库信息，并支持可选的脱敏处理。
- 在启用时，通过可配置的 `mystery_url` 和 `cookies` 支持基于外部库的资料扩展。

增强点：

- 重构 `BoxPlugin`，将数据与渲染逻辑委托给 `BoxService`，简化命令处理，并改进撤回任务管理。
- 通过 `CardTheme` 配置让卡片渲染支持主题化，并调整文本、边框及表情布局行为。
- 精简配置处理流程，增加插件选项并提供更清晰的日志信息，同时新增 `library_switch` 辅助方法。
- 通过使用 `At` 组件和原始消息文本改进提及解析，实现更健壮的目标 ID 提取。

## v1.3.1

New Features:

- 为 box 命令新增别名 `"box"`，以便更轻松地调用。

Bug Fixes:

- 通过集中、健壮的生日解析逻辑修复生日、星座和黄道十二宫字段的计算，并优雅地处理无效日期。

Enhancements:

- 修改库客户端的初始化方式以使用其默认配置。
- 将用户信息查询默认设置为发送图片，除非显式禁用。
- 将生成的 box 卡片图片存储在插件专用的临时目录中，而不是持久化缓存目录，并移除在终止时清理缓存的逻辑。

## v1.3.0

为查询和渲染 QQ 个人资料卡片新增可由 LLM 调用的工具，并将 box 工作流重构为更清晰的数据、发送和召回层。

New Features:

- 暴露一个供 LLM 使用的工具入口，用于查询 QQ 用户资料信息，并可选支持图片生成。
- 引入 `BoxResult` 容器，用于统一 box 查询结果，支持文本和图片两种响应形式。

Enhancements:

- 将 box 命令流程重构为独立的数据获取、渲染、发送和召回职责，并提供更清晰的错误处理和配置检查。
- 优化自动入群/退群处理逻辑，使其使用新的核心 box 信息管线以及事件拦截/停止机制。
- 简化并增强生日、星座、生肖和地址等计算字段的生成逻辑。
- 调整渲染图片的缓存键格式，使其与群 ID 解耦，并简化缓存清理与日志记录。
